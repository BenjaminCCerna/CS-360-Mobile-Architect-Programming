# CS-360-Mobile-Architect-Programming
The mobile app I developed was designed to help users easily create and manage events, with the main goal of providing a simple, user‑centered experience. To support this, I built key screens such as the login page, home dashboard, event creation form, and event list. Each screen followed clear layouts, consistent navigation, and intuitive design choices to keep users in mind and ensure the interface was easy to understand. When coding the app, I used a structured, incremental approach—building one feature at a time, testing frequently, and following Android best practices. I tested the app manually by checking navigation flows, validating inputs, and confirming that data persisted correctly, which helped identify issues early and ensure the app functioned as intended. Throughout development, I had to innovate when handling data persistence and UI responsiveness, adjusting my approach to maintain a smooth user experience. I was particularly successful in the event creation workflow, where I combined UI design, validation, and data handling to create a reliable and user‑friendly feature that demonstrated my skills in mobile development and user‑centered design.
